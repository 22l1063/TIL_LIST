# プロパティ昇格を使用したコードの書き方。

プロパティ昇格について解説する。

## 実行
```
$user = new User("Taro", 20);

echo $user->name; 
```

##　プロパティ昇格のやつ

```
class User
{
    public function __construct(public string $name,private int $age)
    {
    }
}


```

これだとコンストラクタに何も書いていないのに動作する理由がわからない。
しかし、 これはプロパティ昇格という新しい機能？のようなもので、PHP内では以下のように受け取っている。

## PHPが裏で受けっとっている形

```
class User
{
    public string $name;
    private int $age;

    public function __construct(string $name, int $age)
    {
        $this->name = $name;
        $this->age = $age;
    }
}

```
PHPではこのような処理をしている。

`public function __construct(string $name, int $age)`は次のようになる。
`public function __construct(Taro, 20)` とこのようになる。そして、

```
$this->name = $name;
$this->age = $age;
```

```
$this->name = Taro;
$this->age = 20;
```
ちなみに、`$this->name`などは何なのかというと
オブジェクトが持っている name, age というプロパティのことである。
`$user = new User("Taro", 20);`としたときに、

$user
 ├─ name = "Taro"
 └─ age  = 20
このようにプロパティが作成される。



とこのようになる。
実行結果は、`"Taro"`だけだ。
なぜなら、ageはprivateと設定しているからだ。

## なぜ省略できるのか
省略できる理由はコンストラクタの中にアクセス修飾子（public / protected / private）がついているからである。
